# Story 2.1 补充设计：Plugin Snapshot 增强

**版本**：1.0  
**日期**：2026-04-09  
**前置文档**：2026-04-07-story-2.1-template-design.md

---

## 1. 问题描述

当前 Template snapshot 机制存在以下缺陷，导致使用 Template 启动容器后 Claude Code 无法加载已安装的插件：

| # | 问题 | 原因 |
|---|------|------|
| 1 | 插件代码未进入 snapshot | `plugins/cache` 在排除列表中，但这正是插件实际代码的位置 |
| 2 | `installed_plugins.json` 中的路径是宿主机绝对路径 | 如 `/home/njin/.claude/plugins/cache/...`，容器内路径不同 |
| 3 | `settings.json` 中的 hooks 含宿主机绝对路径 | 容器内路径不存在，需要过滤 |

## 2. 设计方案

### 2.1 排除逻辑变更

修改 `config-snapshot.ts` 中的排除规则：

| 路径 | 之前 | 之后 | 原因 |
|------|------|------|------|
| `plugins/cache` | 排除 | **包含** | 插件实际代码所在，必须包含 |
| `plugins/marketplaces` | 包含 | **排除** | 含 .git 大量数据，仅 marketplace 索引，非运行必需 |
| `plugins/data` | 包含 | **排除** | 插件运行时数据，每个 session 独立 |
| `.git`（任意深度） | 无处理 | **排除** | 防止 snapshot 膨胀 |

### 2.2 settings.json 过滤

新增 `sanitizeSettings()` 函数，在 snapshot 阶段对 `settings.json` 进行过滤：

**移除的字段**：
- `hooks` — 包含宿主机绝对路径，容器内无效

**保留的字段**：
- `enabledPlugins` — 插件启用列表，Claude Code 加载插件的前提
- `extraKnownMarketplaces` — 自定义 marketplace 源
- `model` — 用户模型偏好
- `permissions` — 权限配置
- `env` — 环境变量配置
- 其他所有字段原样保留

### 2.3 installed_plugins.json 路径重写

新增 `rewritePluginPaths()` 函数，在 **注入阶段**（template-injector）重写 `installPath`：

```
原始: /home/njin/.claude/plugins/cache/superpowers-marketplace/superpowers/4.3.1
提取: plugins/cache/superpowers-marketplace/superpowers/4.3.1
重写: <CLAUDE_HOME_CONTAINER_PATH>/.claude/plugins/cache/superpowers-marketplace/superpowers/4.3.1
```

重写策略：从 `installPath` 中提取 `.claude/` 之后的相对路径，拼接上容器内的 home 路径。

### 2.4 known_marketplaces.json 路径重写

同样需要重写 `installLocation` 字段。但由于 `plugins/marketplaces` 被排除（不包含在 snapshot 中），将 `installLocation` 清空或移除，让 Claude Code 自行初始化。

## 3. 数据流

```
创建 Template 时 (config-snapshot.ts):
  ~/.claude/settings.json → sanitizeSettings() → snapshot.files["settings.json"]
  ~/.claude/plugins/cache/...  → snapshot.files (包含完整插件代码)
  ~/.claude/plugins/installed_plugins.json → snapshot.files (原样捕获)
  ~/.claude/plugins/known_marketplaces.json → snapshot.files (原样捕获)
  ~/.claude/plugins/blocklist.json → snapshot.files (原样捕获)
  ~/.claude/plugins/install-counts-cache.json → snapshot.files (原样捕获)
  
  排除:
  ~/.claude/plugins/cache 目录 → 不再排除，纳入 snapshot
  ~/.claude/plugins/marketplaces/ → 排除（含 .git）
  ~/.claude/plugins/data/ → 排除（运行时数据）

启动容器时 (template-injector.ts):
  snapshot.files → 逐个写入 verseHome/.claude/
  installed_plugins.json → rewritePluginPaths() 重写路径后写入
  known_marketplaces.json → rewriteMarketplacePaths() 重写路径后写入
```

## 4. 文件改动清单

- 从 `EXCLUDED_DIRS` 移除 `plugins/cache`
- 新增 `plugins/marketplaces`、`plugins/data` 到 `EXCLUDED_DIRS`
- 新增 `.git` 目录排除（`isExcludedPath` 中检查路径段）
- 新增 `sanitizeSettings()` 函数：过滤 settings.json 中的 hooks 字段

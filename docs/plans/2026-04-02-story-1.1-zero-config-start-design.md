## 2. 技术方案

### 2.1 整体流程

```
用户执行 multiverse start
  ↓
CLI (Commander.js) 解析命令
  ↓
Core: 检测 Docker 可用性
  ↓ 失败则友好报错退出
Core: 构建/检查镜像 (自建 Dockerfile)
  ↓
Core: 创建容器 (Dockerode)
  - 挂载 cwd → /workspace (rw)
  - 挂载凭证文件 → /home/coder/.claude/* (ro)
  - 设置入口为 claude-code
  ↓
Core: 启动容器
  ↓
Core: container.attach() 获取 stream
  - 绑定 process.stdin → container.stdin
  - 绑定 container.stdout → process.stdout
  - 处理 SIGWINCH (终端 resize)
  ↓
用户与 claude-code 交互
  ↓
用户退出 claude-code
  ↓
Core: 检测容器退出
  ↓
Core: 移除容器 (autoRemove 或手动 remove)
  ↓
CLI 退出
```

### 2.2 Dockerfile 设计

**位置**：`packages/core/docker/Dockerfile`

```dockerfile
FROM node:22-slim
RUN npm install -g @anthropic-ai/claude-code
RUN useradd -m -s /bin/bash coder
USER coder
WORKDIR /workspace
ENTRYPOINT ["claude"]
```

**关键点**：
- 基于 `node:22-slim`，轻量且满足 claude-code 依赖
- 非 root 用户运行（安全要求）
- 入口为 `claude` 命令（claude-code 的 CLI 名称）
- 镜像 tag：`multiverse/claude-code:latest`

### 2.4 凭证注入设计

**问题**：`~/.claude` 包含认证凭证和运行时状态（对话历史、设置等）。直接挂载会导致不同 verse 的运行时状态互相污染。

**方案：隔离运行时 + 多种凭证来源支持**

Claude Code 支持两种鉴权方式：
1. **文件凭证**：`~/.claude/credentials.json` 等文件
2. **环境变量**：`ANTHROPIC_API_KEY`、`ANTHROPIC_*`、`CLAUDE_CODE_*` 等环境变量

```
宿主机:
~/.claude/
├── credentials.json          ← 认证凭证（需要共享）
├── settings.json             ← 用户设置
├── projects/                 ← 运行时状态（不共享）
└── ...

环境变量:
ANTHROPIC_API_KEY=sk-ant-...  ← 也可能通过环境变量鉴权
CLAUDE_CODE_DEBUG=true        ← Claude Code 配置

容器内:
/home/coder/.claude/
├── credentials.json          ← 只读挂载自宿主机（仅凭证文件）
└── (其他由容器内 claude-code 自动创建)

环境变量:
ANTHROPIC_API_KEY=sk-ant-...  ← 从宿主机环境变量传递
CLAUDE_CODE_DEBUG=true        ← 从宿主机环境变量传递
```

**实现**：
1. **文件凭证**：`CredentialResolver` 扫描 `~/.claude` 目录，识别凭证文件（如 `credentials.json`、`.credentials` 等），只读挂载到容器
2. **环境变量凭证**：`CredentialResolver` 扫描宿主机环境变量，识别 `ANTHROPIC_*` 和 `CLAUDE_CODE_*` 前缀的变量，传递到容器环境
3. 容器内的 `~/.claude` 其余部分是容器自己的临时文件系统，容器退出时随之清理
4. 如果两种凭证都不存在，给出友好提示让用户先运行 `claude login` 或设置 `ANTHROPIC_API_KEY`

**凭证优先级**：
- Claude Code 会按以下顺序查找凭证：
  1. 环境变量 `ANTHROPIC_API_KEY`
  2. 文件 `~/.claude/credentials.json`
- 我们同时支持两种方式，让 Claude Code 自己决定使用哪个
- `CLAUDE_CODE_*` 环境变量用于 Claude Code 的其他配置（如 debug 模式等）

**优点**：
- 不同 verse 的容器完全隔离
- 支持多种鉴权方式（文件 + 环境变量）
- 支持 Claude Code 的所有配置环境变量
- 仅共享认证凭证，避免运行时状态冲突
- 容器退出后运行时状态自动清理

### 2.5 交互模式实现

**方案：Docker attach 透传**

使用 Dockerode 的 `container.attach()` API：

```typescript
async attach(container: Container, stdio: StdioStreams): Promise<void> {
  const stream = await container.attach({
    stream: true,
    stdin: true,
    stdout: true,
    stderr: true,
    hijack: true,
  });

  // 绑定 stdin/stdout
  process.stdin.pipe(stream);
  stream.pipe(process.stdout);

  // 设置 raw mode（保留终端控制字符）
  if (process.stdin.isTTY) {
    process.stdin.setRawMode(true);
  }

  // 处理终端 resize
  process.stdout.on('resize', () => {
    container.resize({ h: process.stdout.rows, w: process.stdout.columns });
  });

  // 处理 SIGINT/SIGTERM
  const cleanup = () => {
    if (process.stdin.isTTY) {
      process.stdin.setRawMode(false);
    }
    stream.end();
  };
  process.on('SIGINT', cleanup);
  process.on('SIGTERM', cleanup);
}
```

**关键点**：
- `hijack: true` 获取双向流
- `setRawMode(true)` 保留终端控制字符（颜色、光标等）
- 监听 `resize` 事件同步终端尺寸
- 注册信号处理器确保清理

### 2.6 容器清理

**策略**：
1. 创建容器时设置 `HostConfig.AutoRemove: true`，容器停止后自动删除
2. 注册 SIGINT/SIGTERM 处理器，异常退出时也能清理：
   ```typescript
   process.on('SIGINT', async () => {
     await containerManager.remove(container);
     process.exit(130);
   });
   ```
3. 使用 try-finally 确保清理逻辑执行

## 4. 错误处理

### 4.1 Docker 未安装

**检测**：`DockerClient.checkAvailability()` 返回 `available: false`

**提示**：
```
❌ Docker is not available.

Please install Docker:
  - Linux: https://docs.docker.com/engine/install/
  - macOS: https://docs.docker.com/desktop/install/mac-install/
  - Windows: https://docs.docker.com/desktop/install/windows-install/

After installation, run: multiverse start
```

### 4.2 凭证未找到

**检测**：`CredentialResolver.resolveCredentialPaths()` 返回空数组且无 `ANTHROPIC_*` 环境变量

**提示**：
```
❌ Claude credentials not found.

Please authenticate using one of these methods:

1. File-based credentials:
   claude login

2. Environment variable:
   export ANTHROPIC_API_KEY=sk-ant-...

Then run: multiverse start
```

### 4.3 容器启动失败

**检测**：`ContainerManager.createAndStart()` 抛出异常

**提示**：
```
❌ Failed to start container: {error message}

Possible causes:
  - Docker daemon not running
  - Insufficient permissions
  - Port conflict

Try: docker ps
```

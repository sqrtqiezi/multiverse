## 3. Node Sidecar（gui-server）

### 3.1 包结构

```
packages/gui-server/
├── package.json        # bin: "multiverse-gui-server"
├── tsconfig.json
└── src/
    ├── index.ts        # 入口：stdin/stdout JSON-RPC 循环
    ├── rpc-handler.ts  # RPC 方法路由
    └── methods/
        ├── config.ts   # 配置文件读写方法
        └── template.ts # template 相关方法（后续扩展）
```

### 3.2 通信协议

JSON-RPC 2.0 over stdin/stdout，每行一个 JSON 对象。

请求示例：
```json
{"jsonrpc":"2.0","id":1,"method":"config.listFiles","params":{"projectPath":"/home/user/project"}}
```

响应示例：
```json
{"jsonrpc":"2.0","id":1,"result":{"files":[{"path":"CLAUDE.md","type":"agent-config"}]}}
```

### 3.3 MVP RPC 方法

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `config.listFiles` | `{projectPath}` | 配置文件树 | 扫描项目和全局配置 |
| `config.readFile` | `{filePath}` | 文件内容 | 读取配置文件 |
| `config.writeFile` | `{filePath, content}` | 成功/失败 | 保存配置文件 |

### 3.4 配置文件扫描逻辑

`config.listFiles` 返回树形结构：

```
项目配置 (projectPath)
├── CLAUDE.md
└── .claude/
    ├── settings.json
    └── commands/
全局配置 (~/.claude)
├── CLAUDE.md
├── settings.json
└── commands/
Template 配置 (~/.multiverse/templates/)
└── [已有的 templates]
```

## 4. 前端设计

### 4.1 主布局

```
┌──────────────────────────────────────────────────────┐
│  Multiverse                              [─] [□] [×] │
├──────────┬───────────────────────────────────────────┤
│          │  CLAUDE.md                    [保存] [撤销] │
│ 配置树    │ ┌───────────────────────────────────────┐ │
│          │ │                                       │ │
│ ▼ 项目    │ │   Monaco Editor                      │ │
│   CLAUDE │ │                                       │ │
│ ▼ 全局    │ │                                       │ │
│   CLAUDE │ │                                       │ │
│ ▼ Template│ │                                       │ │
│   默认配置│ │                                       │ │
│          │ └───────────────────────────────────────┘ │
├──────────┴───────────────────────────────────────────┤
│  状态栏: 已保存 | UTF-8 | Markdown                    │
└──────────────────────────────────────────────────────┘
```

左右分栏使用 Shadcn/ui 的 ResizablePanel，可拖拽调整宽度。

### 4.2 组件结构

```
App
├── AppLayout (ResizablePanel)
│   ├── Sidebar
│   │   └── ConfigTree (树形导航)
│   │       ├── TreeGroup "项目配置"
│   │       ├── TreeGroup "全局配置"
│   │       └── TreeGroup "Templates"
│   └── EditorPanel
│       ├── EditorToolbar (文件名 + 保存/撤销按钮)
│       ├── MonacoEditor (代码编辑器)
│       └── StatusBar (保存状态 + 编码 + 语言)
```

### 4.3 Shadcn/ui 组件使用

- `ResizablePanel` — 左右分栏，可拖拽调整
- `ScrollArea` — 配置树滚动
- `Button` — 工具栏按钮
- `Toaster` — 保存成功/失败通知

### 4.4 Monaco Editor 配置

- 语言：Markdown（CLAUDE.md）、JSON（settings.json）、自动检测
- 主题：跟随系统暗/亮模式
- 功能：语法高亮、自动补全、行号、minimap

## 5. Rust 后端

### 5.1 极薄设计

Rust 后端只负责：
1. 管理 sidecar 进程生命周期
2. 提供通用 `rpc_call` Tauri command 转发请求

```rust
#[tauri::command]
async fn rpc_call(method: String, params: serde_json::Value) -> Result<serde_json::Value, String> {
    // 发送 JSON-RPC 请求到 sidecar stdin
    // 读取 sidecar stdout 响应
}
```

### 5.2 Tauri 配置

在 `tauri.conf.json` 中声明 sidecar：
```json
{
  "bundle": {
    "externalBin": ["binaries/multiverse-gui-server"]
  }
}
```

## 7. 错误处理

| 场景 | 处理方式 |
|------|---------|
| sidecar 启动失败 | GUI 显示友好错误提示，建议检查 Node.js 安装 |
| 配置文件不存在 | 配置树不显示该项，编辑器显示空状态 |
| 文件保存失败 | Toast 提示保存失败及原因 |
| sidecar 进程崩溃 | 自动重启 sidecar，前端显示重连中 |

### 边界情况

- **并发编辑**：暂不处理，后续可加 file watcher
- **无项目目录**：只显示全局配置
- **首次使用**：无 CLAUDE.md 时配置树显示"创建"按钮

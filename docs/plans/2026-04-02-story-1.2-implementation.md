# Story 1.2 Verse Auto-Creation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `multiverse start` 中自动创建并维护基于当前 Git branch 的 verse JSON，记录每次 run 的开始与结束信息。

**Architecture:** 采用 `core` 中的 `BranchResolver + VerseRepository + VerseService` 三层结构。CLI `start` 只编排流程，不直接读写 JSON。Verse 持久化使用“文件锁 + 原子重命名”保证并发与崩溃安全，并在文件命名中加入 branch hash 避免冲突。

**Tech Stack:** TypeScript (ESM), Vitest, Cucumber.js (Gherkin), Node.js fs/promises, child_process (git)

## Fixed Decisions For Corner Cases

- 文件名规则：`<sanitize-lower-branch>__<shortHash(branch)>.json`，避免清洗碰撞和大小写碰撞。
- detached head：文件名使用 `detached-<shortCommit>__<shortHash(fullRef)>.json`。
- run start 时机：`container.createAndStart` 成功后立即写入 `appendRunStart`。
- 锁超时策略：3 秒超时即失败，不抢锁。
- 锁回收策略：锁文件写入 `pid/host/createdAt`；发现进程不存在且锁超过阈值后清理并重试一次。
- schema 预留：主文件写入 `schemaVersion: 1`。

- [ ] **Step 2: 实现文件名计算**

```ts
export function toVerseFileName(branch: string): string {
  const normalized = branch.toLowerCase().replace(/[^a-z0-9._-]/g, '_');
  const digest = createHash('sha1').update(branch).digest('hex').slice(0, 8);
  return `${normalized}__${digest}.json`;
}
```

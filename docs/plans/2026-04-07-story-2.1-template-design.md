## 1. 概述

### 1.1 目标

实现 Template 的创建和使用功能，允许用户：
- 从全局配置创建可复用的 Template
- 使用 Template 启动容器
- 管理多个 Template
- 自动处理 Template 冲突

### 1.2 用户故事

**作为**开发者  
**我想要**保存当前配置为 template，在其他项目复用  
**以便**快速启动一致的开发环境

### 1.3 验收标准

- 执行 `multiverse template create <name>` 创建 template
- Template 包含当前所有配置的快照（CLAUDE.md + .claude/ 目录）
- 执行 `multiverse start --template <name>` 使用 template
- Template 存储在 ~/.multiverse/templates/
- Template 冲突时提供交互式选择
- 首次运行时交互式引导创建或选择 Template

## 3. 数据结构设计

### 3.1 Template 结构

```typescript
interface Template {
  id: string;                    // UUID v4
  name: string;                  // 用户友好名称（唯一）
  description?: string;          // 可选描述
  snapshot: ConfigSnapshot;      // 配置快照
  createdAt: string;            // ISO 8601 格式
}

interface ConfigSnapshot {
  claudeMd?: string;            // CLAUDE.md 内容（可选）
  files: SnapshotFile[];        // .claude/ 目录文件
}

interface SnapshotFile {
  path: string;                 // 相对路径（如 "skills/my-skill.md"）
  content: string;              // 文件内容（Base64 或原文）
}
```

**存储格式**：
- 文件路径：`~/.multiverse/templates/<uuid>.json`
- 文件内容：JSON 格式，包含完整的 Template 对象

### 3.2 Verse v3 结构

```typescript
interface Verse {
  schemaVersion: 3;
  id: string;
  branch: string;
  projectRoot: string;
  templateId: string;           // 新增：关联的 Template ID
  environment: VerseEnvironment;
  createdAt: string;
  updatedAt: string;
  runs: RunRecord[];
}
```

**迁移策略**：
- 检测到 v2 Verse 时，自动创建默认 Template
- 默认 Template 命名为 `auto-migrated-<timestamp>`
- 从当前 `~/.claude/` 创建快照
- 更新 Verse 为 v3，添加 templateId

## 4. CLI 命令设计

### 4.1 创建 Template

```bash
multiverse template create <name> [options]

Options:
  --description <desc>  Template 描述
  --help               显示帮助信息

Examples:
  multiverse template create "我的开发环境"
  multiverse template create "React Setup" --description "React + TypeScript + Vite"
```

**行为**：
1. 检查 `~/.claude/` 目录是否存在
2. 读取 CLAUDE.md（如果存在）
3. 递归读取 `.claude/` 目录所有文件
4. 创建配置快照
5. 验证格式（Markdown、路径合法性）
6. 生成 UUID，保存到 `~/.multiverse/templates/<uuid>.json`
7. 输出成功消息和 Template ID

**错误处理**：
- `~/.claude/` 不存在：提示用户先配置全局环境
- 名称已存在：提示用户使用不同名称或删除旧 Template
- 文件读取失败：显示具体错误和文件路径

### 4.2 列出 Templates

```bash
multiverse template list [options]

Options:
  --json               以 JSON 格式输出
  --help               显示帮助信息

Examples:
  multiverse template list
  multiverse template list --json
```

**输出格式**（表格）：
```
ID                                    Name              Created At
--------------------------------------------------------------------
a1b2c3d4-...                         我的开发环境       2026-04-07 10:30
e5f6g7h8-...                         React Setup       2026-04-06 15:20
```

### 4.3 使用 Template 启动

```bash
multiverse start [options]

Options:
  --template <name>    使用指定的 Template
  --help               显示帮助信息

Examples:
  multiverse start --template "我的开发环境"
  multiverse start --template "React Setup"
```

**行为**：
1. 如果指定 `--template`，加载对应 Template
2. 如果未指定且当前 branch 无 Verse，进入首次运行引导
3. 如果未指定且当前 branch 有 Verse，使用 Verse 关联的 Template
4. 验证 Template 完整性
5. 检查 Template 冲突
6. 创建或更新 Verse
7. 注入配置到容器
8. 启动容器

## 5. 验证策略

### 5.1 创建时验证（格式检查）

**目标**：确保 Template 可以被正确存储和读取

**检查项**：
- CLAUDE.md 是否为有效的 UTF-8 文本
- 文件路径是否合法（无 `../`、绝对路径等）
- 文件内容是否可读
- JSON 结构是否完整（必需字段存在）

**实现**：
```typescript
class TemplateValidator {
  validateFormat(template: Template): ValidationResult {
    // 检查必需字段
    if (!template.id || !template.name || !template.snapshot) {
      return { valid: false, error: 'Missing required fields' };
    }
    
    // 检查文件路径
    for (const file of template.snapshot.files) {
      if (file.path.includes('..') || path.isAbsolute(file.path)) {
        return { valid: false, error: `Invalid path: ${file.path}` };
      }
    }
    
    return { valid: true };
  }
}
```

### 5.2 使用时验证（完整性检查）

**目标**：确保 Template 可以被正确应用到容器

**检查项**：
- Template 文件是否存在
- 所有引用的文件内容是否完整
- 配置内容是否可解析（如 JSON、YAML）
- 文件大小是否合理（防止超大文件）

**实现**：
```typescript
class TemplateValidator {
  async validateIntegrity(template: Template): Promise<ValidationResult> {
    // 检查快照完整性
    if (!template.snapshot.files || template.snapshot.files.length === 0) {
      return { valid: false, error: 'Empty snapshot' };
    }
    
    // 检查文件内容
    for (const file of template.snapshot.files) {
      if (!file.content || file.content.length === 0) {
        return { valid: false, error: `Empty file: ${file.path}` };
      }
      
      // 检查文件大小（限制 10MB）
      if (file.content.length > 10 * 1024 * 1024) {
        return { valid: false, error: `File too large: ${file.path}` };
      }
    }
    
    return { valid: true };
  }
}
```

## 6. 冲突处理流程

### 6.1 场景

当用户执行 `multiverse start --template X` 时，如果当前 branch 已有 Verse 且 `verse.templateId !== X`，则触发冲突。

### 6.2 交互流程

```
检测到 Template 冲突

当前 branch 'main' 已关联 Template 'default'
你想使用 Template 'my-dev-env'

请选择：
1. 创建新 branch 'main-my-dev-env' 并使用新 Template
2. 归档当前 Verse 并在此 branch 使用新 Template
3. 取消操作

你的选择 [1-3]: _
```

### 6.3 选项处理

**选项 1：创建新 branch**
1. 生成建议的 branch 名称：`<current-branch>-<template-name-slug>`
2. 询问用户确认或自定义名称
3. 执行 `git checkout -b <new-branch>`
4. 创建新 Verse，关联新 Template
5. 继续启动流程

**选项 2：归档当前 Verse**
1. 将当前 Verse 的 `status` 设置为 `archived`
2. 在当前 branch 创建新 Verse，关联新 Template
3. 继续启动流程

**选项 3：取消操作**
1. 输出提示信息
2. 退出程序（exit code 0）

## 7. 首次运行引导

### 7.1 触发条件

当用户执行 `multiverse start` 且：
- 未指定 `--template` 参数
- 当前 branch 没有关联的 Verse

### 7.2 交互流程

```
欢迎使用 Multiverse！

当前 branch 'main' 还没有关联 Template

请选择：
1. 使用当前全局配置创建默认 Template
2. 从现有 Template 中选择
3. 创建新的命名 Template

你的选择 [1-3]: _
```

### 7.3 选项处理

**选项 1：创建默认 Template**
1. 从 `~/.claude/` 创建快照
2. 命名为 `default-<timestamp>`
3. 创建 Verse，关联此 Template
4. 继续启动流程

**选项 2：从现有 Template 选择**
1. 列出所有可用的 Templates
2. 用户输入编号或名称
3. 创建 Verse，关联选中的 Template
4. 继续启动流程

**选项 3：创建新 Template**
1. 询问 Template 名称
2. 可选：询问描述
3. 从 `~/.claude/` 创建快照
4. 创建 Verse，关联此 Template
5. 继续启动流程

## 8. 错误处理

### 8.1 Template 不存在

**场景**：用户指定的 Template 名称不存在

**处理**：
```
错误：Template 'my-template' 不存在

可用的 Templates：
1. default
2. React Setup
3. 我的开发环境

提示：使用 'multiverse template list' 查看所有 Templates
```

### 8.2 Template 格式错误

**场景**：Template 文件损坏或格式不正确

**处理**：
```
错误：Template 'my-template' 格式错误

详细信息：
  文件：~/.multiverse/templates/a1b2c3d4-....json
  错误：Invalid JSON at line 15, column 3

建议：
1. 使用 'multiverse template create' 重新创建
2. 手动修复 JSON 文件
3. 删除损坏的 Template
```

### 8.3 配置文件缺失

**场景**：`~/.claude/` 目录不存在或为空

**处理**：
```
错误：全局配置目录不存在

路径：~/.claude/

建议：
1. 运行 'claude-code' 初始化全局配置
2. 手动创建 ~/.claude/ 目录和配置文件
3. 从其他机器复制配置文件
```

### 8.4 迁移失败

**场景**：Verse v2 → v3 迁移失败

**处理**：
```
警告：Verse 迁移失败

详细信息：
  Verse ID：abc123
  Branch：main
  错误：Failed to create default template

当前 Verse 保持不变，未做任何修改

建议：
1. 检查 ~/.claude/ 目录是否存在
2. 手动创建 Template 后重试
3. 查看日志文件：~/.multiverse/logs/migration.log
```

## 11. 性能考虑

### 11.1 配置快照大小

**问题**：.claude/ 目录可能包含大量文件

**优化**：
- 限制单个文件大小（10MB）
- 限制总快照大小（50MB）
- 排除不必要的文件（.DS_Store、node_modules 等）

### 11.2 Template 加载速度

**问题**：大型 Template 加载慢

**优化**：
- 延迟加载（只在需要时读取文件内容）
- 缓存常用 Template
- 压缩存储（gzip）

### 11.3 迁移性能

**问题**：大量 Verse 迁移耗时

**优化**：
- 按需迁移（读取时迁移，而非批量迁移）
- 并行迁移（多个 Verse 同时迁移）
- 进度提示（显示迁移进度）

## 12. 安全考虑

### 12.1 路径遍历攻击

**风险**：恶意 Template 包含 `../` 路径

**防护**：
- 验证所有文件路径
- 禁止绝对路径
- 禁止 `../` 等相对路径

### 12.2 文件大小攻击

**风险**：恶意 Template 包含超大文件

**防护**：
- 限制单个文件大小（10MB）
- 限制总快照大小（50MB）
- 验证时检查文件大小

### 12.3 敏感信息泄露

**风险**：Template 可能包含敏感信息（API keys、密码等）

**防护**：
- 创建时警告用户检查敏感信息
- 提供 `.templateignore` 文件排除敏感文件
- 文档中明确说明安全最佳实践

## 15. 总结

### 15.1 关键决策

1. **Template 作为 Verse 必需字段**：确保每个 Verse 都有明确的配置来源
2. **从全局配置创建 Template**：简化用户操作，避免配置分散
3. **渐进式实现**：降低风险，每步都可测试
4. **交互式引导**：提升用户体验，减少错误操作
5. **双重验证**：创建时验证格式，使用时验证完整性

### 15.2 风险和缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 迁移失败 | 高 | 保留原 Verse，提供详细错误信息 |
| Template 冲突 | 中 | 交互式引导，提供多种选择 |
| 配置快照过大 | 中 | 限制文件大小，排除不必要文件 |
| 路径遍历攻击 | 高 | 严格验证路径，禁止相对路径 |

### 15.3 成功指标

- 用户可以在 5 分钟内创建和使用 Template
- Template 冲突处理成功率 > 95%
- Verse 迁移成功率 > 99%
- 单元测试覆盖率 > 80%
- E2E 测试覆盖核心流程
